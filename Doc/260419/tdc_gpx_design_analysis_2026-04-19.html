1. 개요 및 설계 목표

TDC-GPX Controller 는 ams TDC-GPX 고정밀 시간 디지털 변환기 (4 chip × 8 stop) 를 FPGA 에서 제어하면서 측정 결과를 VDMA 를 통해 DDR 로 전달하는 파이프라인이다. Laser LiDAR 용 전용 설계로, 측정 거리에 따라 쓰루풋/셀 크기를 런타임 조절하여 PRF 상한까지 활용한다.

핵심 설계 목표

  • TDC-GPX I-mode — 외부 stop 이벤트 → IFIFO 기반 raw word 수집
  • 4 chip 병렬 제어 — 독립 bus_phy + chip_ctrl 로 chip 간 격리
  • Rise/Fall slope 분리 파이프라인 — 2× 독립 cell/face 조립 경로
  • AXI-Stream 기반 모듈화 — 모든 데이터 경로 handshake 기반, skid 버퍼/xpm_fifo_axis 로 backpressure 흡수
  • Face-단위 VDMA 프레임 생성 — 48-byte header + chip×stop×beats 순차 출력
  • 거리 기반 타이밍 규약 — shot_period = 1.5 × round-trip (50% PRF headroom)
설계 스케일 (tdc_gpx_pkg.vhd): 4 chips, 8 stops/chip, max 7 hits/stop, 17-bit hit slot, TDATA 32 또는 64-bit 선택, Rise+Fall 2 slope → 총 8 개 cell_builder, 2 face_assembler + 2 header_inserter.

4. Cluster 1 — TDC 버스 제어

tdc_gpx_chip_ctrl (coordinator)

역할: 3 개의 서브 FSM (chip_init, chip_run, chip_reg) 를 phase 기반으로 조정하고, 버스 request mux + response routing + raw-out FIFO 를 담당.

Coordinator FSM (6 phase)

Phase역할진입 조건
PH_INITchip_init 활성 (powerup/cfg_write)reset 해제, soft_reset, drain_to_init
PH_IDLE대기init 완료
PH_RUNchip_run 활성i_cmd_start
PH_REGchip_reg 활성i_cmd_reg_read/write
PH_CFG_WRITEchip_init 의 cfg_write 경로i_cmd_cfg_write
PH_RESP_DRAINstale bus response 배출 (quarantine 포함)모든 sub-FSM 완료 후 defensive 진입

하드 cap 및 자가 복구

  • PH_RESP_DRAIN 정상 종료: bus idle 3 cycle 유지
  • Hard cap: 15 cycle 도달 시 bus 가 여전히 busy 면 quarantine
  • Secondary watchdog: s_drain_quarantine_cnt_r (16-bit) 포화 시 s_err_bus_fatal_r 세움
  • Auto-recover (Round 13 follow-up): fatal 세팅 후 bus 가 g_BUS_IDLE_STABLE_CLKS (기본 4096 cy ≈ 20 µs) 연속 idle 유지 시 자동 PH_INIT 로 escape, s_err_force_reinit_r 스티키 세움
  • SW 우회: i_cmd_force_reinit (최우선) — 외부에서 버스 flush 했다는 전제 하에 PH_INIT 직행

Raw AXI-Stream 출력 FIFO

  • Depth 6, 타입 t_raw_entry {valid, tdata[31:0], tuser[7:0], drain}
  • Backpressure 등록 1 cycle 지연 흡수 + control beat (drain_done, ififo1_done) 예약
  • Data-sacrifice 정책 (Round 13 follow-up P1): control beat 가 FIFO full 일 때 가장 오래된 data beat 를 evict 하여 control 을 반드시 enqueue
  • TUser(5) 는 faulted 플래그 (drain_done_faulted 전파용, audit 4번)

tdc_gpx_chip_init / chip_run / chip_reg

모듈FSM 상태 개수주요 기능타임아웃
chip_init10 PuResN 펄스 (~240 ns) → 레지스터 쓰기 (REG0-7) → ALU trigger → 설정 완료 bus 응답 65535 cy
chip_run13 ARMED → CAPTURE → DRAIN_LATCH/EF1/EF2/BURST/FLUSH → ALU_PULSE → ALU_RECOVERY per-state 65535 cy watchdog

chip_run 의 drain 완료 의미 분리 (Round 13 axis 1a, follow-up 4번):
  • o_drain_done = 정상 완료 1-clk 펄스
  • o_drain_done_faulted = 동일 cycle 에 co-assert (fallback 경로에서 expected-vs-actual count 불일치 감지)
  • 이 faulted 플래그가 raw stream 의 tuser(5) 에 실려 cell_builder 까지 전달되고, 최종적으로 o_m_axis_tuser(0) 로 face_assembler 에 도달 (P1 rework)

tdc_gpx_bus_phy

역할: TDC 칩 버스 프로토콜 (R/W, tick 기반) 구현 및 응답 AXI-Stream 생성.

FSM

타이밍

  • 3-bit tick counter (s_tick_r) 로 Phase A → L → H 분할 (1 + N-2 + 1)
  • 최소 i_bus_ticks = 4 → req_valid → rsp_valid 지연 = N+1 clocks (tdc 도메인)
  • 응답은 1-stage holding register (no skid; chip_ctrl 가 next req 전에 소비)

tdc_gpx_stop_cfg_decode (P1 rework 후)

역할: echo_receiver → expected IFIFO drain counts 디코드 + 거리 기반 orphan window.

거리 기반 shot window (Round 13 follow-up 5번)

shot_start(T0) ─▶ R = max_range_clks ─▶ W = R + margin ─▶ T1 = next shot_start
                   (물리적 round-trip 경계)  (latency 흡수)    (shot_period)
                                             │                 │
                                             │ orphan 감지 zone │
  • Snapshot: s_max_range_snap_r (shot_start 시점 latch)
  • Counter: s_window_cnt_r (24-bit) → overflow-free
  • Generic: g_WINDOW_MARGIN_CLKS = 32 (기본, 16-bit 상한)
  • max_range_clks = 0 → window 영원히 open (orphan 감지 실질 비활성)

기타 Cluster 1 모듈

  • cmd_arb — 4 chip 에 걸친 reg access 중재; 1-depth overlap queue + dispatch-wait watchdog
  • csr_chip — per-chip AXI4-Lite CSR (8 CTL + 8 STAT)
  • err_handler — ErrFlag 디바운스 + REG12 분류 + per-chip soft_reset + error_fill gating

5. Cluster 2 — Event 디코드

tdc_gpx_decoder_i_mode

28-bit TDC-GPX I-mode raw word 에서 hit/slope/channel/stop_id 추출.
  • 입력: tdata[27:0], tuser[7]=drain_done, tuser[0]=ififo_id, tuser[5]=faulted (control beat 에서만)
  • 출력: tdata[16:0]=raw_hit, tuser[5:0] = slope/cha_code/stop_id_local (data beat) 혹은 faulted flag 보존 (control beat)
  • Pipeline: 1-stage 레지스터드 (always-accept 패턴, 단일 bubble)

Field mapping (I-Mode, SINGLE_SHOT)

Raw bit필드
[27:26]ChaCode (2-bit, IFIFO 내 channel)
[25:18]StartNum (8-bit, SINGLE_SHOT 에서 항상 0)
[17]Slope (rising / falling)
[16:0]Hit (17-bit, BIN 단위 Stop–Start 시간)

tdc_gpx_raw_event_builder

chip_id 태깅 + per-stop hit_seq 카운터 + out-of-range stop_id 검출.
  • 입력: decoder_i_mode 출력 (8-bit tuser)
  • 출력: 16-bit tuser 확장 — slope(0), chip_id[2:1], stop_id[5:3], ififo_id[6], drain_done[7], hit_seq_local[10:8], shot_seq[4:0][15:11]
  • 내부 상태: per-stop hit counter (8 × 3-bit), final drain_done 에 reset
  • Pipeline: 1-stage 레지스터드

Control beat 처리

  • tuser[7]=1, tuser[6]=0: IFIFO1 done (중간) → 카운터 리셋 안 함
  • tuser[7]=1, tuser[6]=1: Final drain_done → 모든 stop hit counter 리셋
  • faulted bit (tuser[5]) 는 control beat 에서만 의미 있고 그대로 패스스루

6. Cluster 3 — Cell 조립

tdc_gpx_cell_pipe (wrapper)

  • 입력: 4 chip × event stream (AXIS, 16-bit tuser)
  • slope_demux (registered) → rise/fall 분리
  • 8 × tdc_gpx_cell_builder (4 rise + 4 fall)
  • 출력: per-chip cell stream (rise+fall 각 4 채널), TUSER 1-bit (faulted)

tdc_gpx_cell_builder (핵심 메모리 집약 모듈)

Event → 셀 (per-stop hit 누적, 출력 serialize). Ping-pong dual buffer 로 연속 shot overlap 흡수.

Dual-Buffer 구조

항목값
버퍼 개수2 (ping-pong)
per-buffer 엔트리8 stops (c_MAX_STOPS_PER_CHIP)
per-entryt_cell (hit_slot×7, hit_valid×7, slope_vec×7, hit_count, hit_dropped)

Collect FSM (p_collect)

  • ST_C_IDLE — shot_start 대기
  • ST_C_ACTIVE — hit 누적; 최종 drain_done 시 BUF_FULL → 다른 버퍼로 전환 또는 DROP
  • ST_C_DROP — 버퍼 부족 시 데이터 흡수 (tready='1') 하지만 버퍼에 쓰지 않음
  • ST_C_QUARANTINE — 65K cy watchdog × c_QUARANTINE_TICK_CAP (4-bit) 반복 → 포기 후 force-escape + post-escape absorb 12-bit

Output FSM (p_output)

  • ST_O_IDLE → ST_O_LOAD → ST_O_ACTIVE (정상, cell 당 1-clk LOAD bubble)
  • ST_O_WAIT_IFIFO2 — 스톱 3→4 경계에서 IFIFO2 미준비 시 stall
  • ST_O_TIMEOUT_EOS — IFIFO2 timeout (x"FFFF") 시 합성 tlast beat (faulted=1)
  • 쓰루풋: cell 당 (beats_per_cell + 1) cycle (LOAD bubble 포함)

tuser(0) = faulted 생성 (P1 rework)

  • Per-buffer s_buf_faulted_r(0:1): final drain_done beat 에 tuser(5)='1' 이면 latch
  • p_output 이 tlast beat 에서만 s_tuser_r(0) = s_buf_faulted_r(v_rd) 로 set, 그 외 beat 는 0
  • ST_O_TIMEOUT_EOS synthetic tlast: tuser 강제 '1' (degraded output)
  • 이로써 faulted 는 data beat 와 동일한 xpm_fifo_axis 경로를 따라 이동 → cross-shot race 원천 차단

7. Cluster 4 — Face/Frame 조립 + 헤더 삽입

tdc_gpx_face_assembler

4 chip × cell stream → 순차 조립된 row. Strict in-order (chip0→1→2→3), inactive chip skip, timeout chip blank-fill.

입력 FIFO (per-chip)

항목값
개수4 × xpm_fifo_axis
Depth16 (distributed RAM)
TDATAg_TDATA_WIDTH (32 또는 64)
TUSER1 (faulted 플래그 전달; P1 rework 후 활용됨)

주요 FSM (ST_IDLE → SCAN → RESOLVE → FORWARD)

  • ST_SCAN: 최저-index active chip 선택; timeout 시 blank-fill
  • ST_RESOLVE: is_last_chip 결정 (1-clk)
  • ST_FORWARD: cell 데이터 출력 (chip-당 1-cell = beats_per_cell × 1-clk)
  • blank-fill: timeout 시 metadata beat[10]=error_fill=1 로 합성

shot_start overrun (Q&A #36)

  • row 처리 중 shot_start 도착 → 현재 row 는 blank-fill 로 완결 + 다음 shot 을 pending latch
  • face 는 중단하지 않고 자연스럽게 다음 shot 으로 전환 (VDMA frame size 유지)

row_done_faulted 판정 (Round 13 axis 1c + P1 rework)

  • zero active_mask → 즉시 row_done + faulted='1' (synthetic row)
  • blank-fill 경로 → faulted='1'
  • partial/blank error mask 가 있으면 faulted='1'
  • chip tlast 시점에 s_in_tuser(i)='1' → pending[i]='1' → row 완료 시 OR-fold

출력 FIFO + header_inserter 경로

  • face_assembler → u_fifo_out (16-deep xpm_fifo_axis)
  • output_stage 레벨 u_face_rise_fifo / u_face_fall_fifo 추가 buffering
  • header_inserter 가 face_start 신호로 48-byte 헤더 prepend

tdc_gpx_header_inserter

FSM

Header ROM

  • s_hdr_rom_r: 12 word × 32-bit = 48 byte
  • face_start 시점에 미리 계산 (frame_id, face_id, shot_seq, geometry, timing, timestamp, chip_error_mask 등)
  • 32-bit TDATA → 12 prefix beats, 64-bit TDATA → 6 prefix beats

drain watchdog (Round 11 item 3)

  • ST_DRAIN_LAST / ST_ABORT_DRAIN 에서 downstream wedge 감지 → watchdog 만료 시 tvalid='0' 강제 복귀 (last beat 유실 허용, liveness 우선)
  • Sticky 로 관찰 (o_err_drain_timeout)

8. 사이드카 모듈

tdc_gpx_face_seq

  • FSM: ST_IDLE → ST_WAIT_SHOT → ST_IN_FACE
  • 카운터: s_face_id_r[7:0], s_global_shot_seq_r[15:0], s_face_shot_cnt_r[15:0], s_frame_abort_cnt_r[15:0]
  • 역할: cmd_start 수락 + shot gating + face/frame boundary 관리 + rise/fall slope 동기화

tdc_gpx_err_handler

  • FSM: ST_IDLE → ST_READ_REG12 → ST_WAIT_READ (65K cy) → ST_RECOVERY → ST_WAIT_RECOVERY (9999 cy) → ST_WAIT_FRAME_DONE (131K cy)
  • Retry counter s_retry_cnt_r (g_MAX_RETRIES 기본 3) → 초과 시 err_fatal
  • ErrFlag 디바운스 + REG12 분류 + per-chip soft_reset + error_fill 데이터 게이팅

12. Backpressure 와 Critical Path

주요 backpressure 경로

  1. VDMA 가 느릴 때: output_stage 의 face_rise/fall_fifo (16-deep) 가 가득 참 → header_inserter ST_DATA 에서 stall → face_assembler 출력 FIFO 포화 → face_asm ST_FORWARD stall → 입력 FIFO 채워짐 → cell_builder 출력 stall → cell_builder 버퍼 free 안됨 → 새 shot_start 수용 못 함 → shot_dropped 발생
  2. cell_builder dual buffer 꽉 참: ping-pong 모두 BUF_SHARED 혹은 COLLECT 인 경우 새 shot_start 는 ST_C_DROP 경로로 — tready='1' 로 흡수하되 버퍼에는 안 씀
  3. chip_ctrl raw FIFO 꽉 참: s_raw_hold_busy='1' → chip_run 의 bus read 를 stall (i_raw_busy) → TDC 버스 idle → shot 누락 가능

Timing-critical 지점 (post-synth 검증 권장)

  • chip_ctrl raw FIFO shift + enqueue 로직 (data-sacrifice 포함): 중첩 for-loop 로 LUT mux 체인이 길어짐. 200 MHz 에서 report_timing_summary 로 slack 확인 필요.
  • cell_builder p_collect: shot_start + drain_done + hit write + quarantine 계산 모두 단일 cycle 내 → 조합 로직 깊이 주의
  • face_assembler ST_SCAN: 4-chip priority encode + ready/timeout/overflow 판정
  • header_inserter ROM build: face_start 1 cycle 안에 12-word ROM 계산

Deadlock 방지 장치

  • chip_run: 7 곳의 s_wait_cnt_r = x"FFFF" watchdog → synthetic drain_done
  • chip_ctrl: PH_RESP_DRAIN hard cap + fatal → auto-recover (g_BUS_IDLE_STABLE_CLKS)
  • cell_builder: QUARANTINE (65K cy × 15 tick = ~4.9 ms) → force-escape + post-escape absorb
  • header_inserter: drain watchdog → tvalid 강제 하강 (beat 유실 허용)
  • face_assembler: shot_cnt hard cap (65535 cy) → blank-fill 강제
